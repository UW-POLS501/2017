The Mathematics of Enigma

QSS: Exercise 6.6.1

When calculating factorials and combinations using the functions lfactorial and lchoose which calculate the natural logarithm of the factorial (\(\log(x!)\)) and binomial coefficient, \[ \log \binom{n}{k} = \log (n!) - \log(k!) - \log ((n - k)!) \] Work with the logarithms of factorials or binomial coefficients since these numbers get large very quickly, and can result in Inf in intermediate calculations. For example,

factorial(1000) / (factorial(998) * factorial(2))
## Warning in factorial(1000): value out of range in 'gammafn'
## Warning in factorial(998): value out of range in 'gammafn'
## [1] NaN

doesn’t work since

factorial(1000)
## Warning in factorial(1000): value out of range in 'gammafn'
## [1] Inf

But this can be calculated using log factorials,

exp(lfactorial(1000) - lfactorial(998) - lfactorial(2))
## [1] 499500

Even better is to use lchoose or choose in R, which are more robust since they were written to handle these sorts of numerical issues,

exp(lchoose(1000, 2))
## [1] 499500
choose(1000, 2)
## [1] 499500

This is a reminder. A good rule of thumb if you don’t want to remember too much numerical methods is to use logarithms whenever you are using really big or really small numbers. Also, if you ever get Inf in a calculation, switch to the log scale.
